Common Issues

Meta Developer Console Setup

  1. App is not in Live Mode - Ensure your app is set to "Live" mode, not "Development" mode.
  2. Missing App Review - Basic Display API requires app review by Meta.
  3. Tester Access - Before approval, you need to add yourself as a tester in the Meta Developer Console.
  4. Incorrect Redirect URI - The redirect URI should be exactly https://srijanmishra08.github.io/reshuffle-oauth/instagram-callback
  5. Missing Business Verification - Some apps require business verification.
  6. Missing Privacy Policy URL - Make sure you've added the privacy policy URL to your app settings.
  7. Instagram User is not Tester - The Instagram account you're using to test must be added as a tester in the app.

iOS App Setup

  1. URL Scheme Configuration - Ensure you have com.srijan.Reshuffle registered in Info.plist under URL Types.
  2. Application Query Schemes - Add instagram to LSApplicationQueriesSchemes in Info.plist.
  3. Callback Handling - Check that the SceneDelegate correctly handles URL callbacks.
  4. Client ID Mismatch - Verify the client ID matches between your app and the Meta Developer Console.

iOS Info.plist Configuration

Make sure your Info.plist includes these entries:

<key>CFBundleURLTypes</key>
<array>
    <dict>
        <key>CFBundleTypeRole</key>
        <string>Editor</string>
        <key>CFBundleURLName</key>
        <string>com.srijan.Reshuffle</string>
        <key>CFBundleURLSchemes</key>
        <array>
            <string>com.srijan.Reshuffle</string>
        </array>
    </dict>
</array>

<key>LSApplicationQueriesSchemes</key>
<array>
    <string>instagram</string>
    <string>fb-messenger-share-api</string>
    <string>fbapi</string>
    <string>fbauth2</string>
</array>

Fix Steps

Current Error: "Invalid app" error indicates that Instagram is not recognizing your app for authentication.

Step 1: Check Tester Status

Make sure your Instagram account is added as a tester in the Meta Developer Console:

  1. Go to Meta Developer Console
  2. Select your app
  3. Navigate to "Roles" in the sidebar
  4. Under "Instagram Testers", add your Instagram account
  5. Accept the tester invitation from your Instagram account

Step 2: Verify Client Settings

In the Meta Developer Console:

  1. Go to Products > Instagram > Basic Display
  2. Verify your "Instagram App ID" matches 583498367953205
  3. Verify "Valid OAuth Redirect URIs" includes https://srijanmishra08.github.io/reshuffle-oauth/instagram-callback
  4. Make sure "Deauthorize Callback URL" and "Data Deletion Request URL" are properly set (or use the same callback URL)

Step 3: Verify App Review Status

Before an app can go live with Instagram API, it needs to be reviewed and approved by Meta:

  1. Go to App Review > Permissions and Features
  2. Check if "instagram_basic" and "instagram_graph_user_profile" permissions are approved
  3. If not approved, submit for review or continue with Development Mode using tester accounts only
Important: For development testing, your Instagram account must be explicitly added as a tester in the Meta Developer Console, and that Instagram account must accept the invitation.
Testing Before Approval: You can test your app before Meta approval by:
  1. Adding your Instagram account as a tester
  2. Using the app in Development Mode
  3. Testing with only approved Instagram test users

Detailed Error Information

The "invalid app" error usually appears when:

  1. The app is trying to access Instagram API from an unauthorized domain or app
  2. The Instagram user is not added as a tester (for apps in development)
  3. The app is missing required settings in the Meta Developer Console
  4. The client ID in your app doesn't match the one in the Developer Console
